# Engineering Philosophy

## 1. Drift는 제거할 수 없다

Dead reckoning 기반 시스템에는 반드시 drift가 발생한다.

해결이 아니라 관리의 문제로 접근해야 한다.

## 2. 센서보다 모델이 중요하다

센서 주기가 낮거나 공백이 있어도 prediction과 모델 기반 보정으로 시스템을 유지할 수 있다.

## 3. 재사용 가능한 구조가 핵심이다

한 번 쓰고 버리는 코드보다 여러 환경에서 재사용 가능한 시스템 설계를 우선한다.

## 4. 문제의 80%는 인프라다

알고리즘보다 더 자주 문제를 일으키는 것은 다음과 같다.

- ROS 설정
- 네트워크
- Docker
- 빌드 환경

## 5. 실제 시스템은 완벽하지 않다

이론과 현실은 다르다.

항상 노이즈, 딜레이, 누락을 전제로 설계한다.
