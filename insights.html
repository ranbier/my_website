Engineering Insights

좋은 추정기 하나가 모든 문제를 해결해 주지는 않습니다.

EKF는 강력한 도구지만 drift를 완전히 없애 주지는 않습니다. 센서 공백도 반드시 생기고, 구조 설계가 빈약하면 좋은 필터를 써도 시스템 전체는 흔들립니다. 그리고 실제 현장 문제는 생각보다 자주 Docker, ROS, 네트워크 같은 환경에서 시작됩니다.

Mistakes

반복되는 실수의 공통점은 시스템 전체 흐름을 놓친다는 데 있습니다.

  • 센서가 항상 정상적으로 들어온다고 가정함
  • 구조 문제보다 튜닝 문제부터 의심함
  • 인프라 문제를 알고리즘 문제로 착각함
  • 좌표계와 reference frame 확인을 뒤로 미룸

이 실수들은 대부분 한 레이어만 보고 시스템을 판단할 때 발생합니다. 그래서 문제가 생기면 다시 전체 흐름으로 돌아가는 습관이 중요합니다.

Patterns

반복해서 통하는 패턴은 언제나 데이터 흐름을 먼저 보는 것입니다.

  1. 문제가 생긴 지점을 추측하기보다 데이터가 실제로 흐르는지 확인합니다.
  2. 센서 -> 상태 -> 제어 순서로 레이어를 하나씩 점검합니다.
  3. 가능하면 시각화와 로그를 같이 사용해 눈으로 확인합니다.
핵심 인사이트

시스템 이해도는 새로운 알고리즘을 아는 것보다, 반복되는 실수와 패턴을 어떻게 정리해 두었는지에서 더 빠르게 쌓이기도 합니다.
